Test the Web Forward content kit

Test the Web Forward is W3C's one stop shop for Open Web Platform testing. In this content kit we will help you teach developers what they need to know to get involved.

Please file any issues you find against this content kit's github repo.

  • Understanding web specifications
  • Try your hand at testing! Fix these broken tests.

Who is this content kit for?

Test the Web Forward is ideally suited to authors with experience working closely with the browser APIs. Their expertise may be broad across a number of areas (e.g. DOM) or specific to one particular specification. Library/polyfill authors in particular have detailed implementation knowledge that would be useful for contributing to interoperability tests.

How can you help?

You may be more ready than you think. If you have:

  • posted a question on stackoverflow asking why browser A behaves differently to browser B
  • raised a bug report for a particular browser
  • written code that polyfills a web specification for one or more browsers
  • blogged, spoken or taught about a web specification
  • created a demo to show off features of a web standard
  • experience writing tests.

Great! You're already thinking critically about interoperability. We could use your help at Test the Web Forward. You can:

  • review the existing tests
  • raise an issue for inconsistent behaviour in browsers
  • write a new test demonstrating bugs you've found
  • port tests you have already written.
